## 1. 정의

데이터 타입은 값을 생성한 이후의 변경 가능성 여부에 따라 두 가지로 나눌 수 있습니다.

* **Immutable(불변)**: 값을 한 번 만들면 그 내용을 **바꿀 수 없는** 데이터 타입  
* **Mutable(가변)**: 값을 만든 후에도 내용을 **바꿀 수 있는** 데이터 타입

---

## 2. 타입별 분류

Python의 주요 데이터 타입들이 어디에 속하는지 한눈에 살펴봅시다.

| 구분 | 데이터 타입 | 예시 | 변경 가능 여부 |
| :--- | :--- | :--- | :--- |
| **Immutable** | `int` (정수), `float` (실수) | `a = 10` | ❌ 불가능 |
| | `str` (문자열) | `s = "hello"` | ❌ 불가능 |
| | `tuple` (튜플) | `t = (1, 2)` | ❌ 불가능 |
| | `frozenset` (고정된 집합) | `fs = frozenset([1])`| ❌ 불가능 |
| **Mutable** | `list` (리스트) | `l = [1, 2, 3]` | ✅ 가능 |
| | `dict` (딕셔너리) | `d = {'key': 'value'}`| ✅ 가능 |
| | `set` (집합) | `s = {1, 2}` | ✅ 가능 |

## 3. 코드로 보는 동작 원리 차이

이 차이는 변수를 다른 변수에 할당하고 값을 변경하려고 할 때 확실하게 드러납니다.

### ❌ Immutable(불변)

여기서는 `int`를 예시로 사용하여 설명하겠습니다.

```python
a = 10
b = a   # b에 a의 값을 할당
b = 20  # b에 새로운 값을 할당

print(a)  # 출력: 10
print(b)  # 출력: 20
```

* **결과:** `b`를 20으로 변경해도 `a`는 그대로 10입니다.
* **이유:** `int`는 불변 객체입니다. `b = 20`을 실행하는 순간, 기존 값을 바꾸는 것이 아니라 **메모리에 새로운 객체(20)를 만들고 `b`가 그것을 가리킵니다.** 따라서 `a`에는 아무런 영향이 없습니다.

---

### ✅ Mutable(가변)

여기서는 `list`를 예시로 사용하여 설명하겠습니다.

```python
a = [1, 2, 3]
b = a         # b가 a와 같은 리스트를 가리킴 (참조 복사)
b.append(4)   # b를 통해 리스트 내용 변경

print(a)      # 출력: [1, 2, 3, 4] ← a의 값도 함께 변경됨
```

* **결과:** `b`에 요소를 추가했는데 `a`의 값도 변했습니다.
* **이유:** `list`는 가변 객체입니다. `b = a`를 하면 두 변수는 **메모리 상의 같은 리스트 객체를 공유(참조)**합니다. 따라서 한쪽에서 내용을 수정하면, 같은 곳을 바라보는 다른 변수에도 변경 사항이 반영됩니다.

---

## 요약

* **Immutable** (`int`, `str`, `tuple` 등)
  * 값이 변해야 한다면, 기존 객체를 수정하는 것이 아니라 **새로운 객체**를 생성하여 재할당합니다.

* **Mutable** (`list`, `dict`, `set` 등)
  * 값이 변할 때, **기존 객체 내부의 값**을 직접 수정합니다. 해당 객체를 참조하는 모든 변수가 영향을 받습니다.
